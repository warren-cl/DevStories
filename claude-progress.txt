=== Session: 2026-03-02 (feat/version-2-proposal branch) ===
Stories: Version 2 proposal — Theme management, Backlog/Breakdown views, Drag-and-drop, Burndown chart, Sort, Set Current Sprint, effort-based progress
Status: in-progress (uncommitted on feat/version-2-proposal, 87 files changed vs main)

Summary:
Major feature branch introducing Themes as a top-level grouping above Epics,
dual view modes (Backlog by sprint / Breakdown by theme→epic→story), drag-and-drop
reordering, sprint burndown chart, story point–based progress tracking, and
several UX improvements.

### New Features

**Theme Management**
- New `Theme` type (src/types/theme.ts): id, title, status, priority, created/updated
- Parser.parseTheme() in parser.ts with gray-matter frontmatter parsing
- Store: themes Map, getTheme(), getThemes(), getEpicsByTheme(), getEpicsWithoutTheme()
- createTheme command with QuickPick for title, duplicate detection, kebab-case filenames
- createThemeUtils.ts: findNextThemeId(), generateThemeMarkdown(), appendEpicToTheme()
- changeStatus extended to handle Theme status updates (updateThemeStatus)
- schemas/theme.schema.json with required id/title/status/created
- validateThemeName() in inputValidation.ts

**Dual View Modes (Breakdown / Backlog)**
- `ViewMode` type ('breakdown' | 'backlog') in storiesProviderUtils.ts
- storiesProvider.ts heavily reworked (~408 lines added):
  - Breakdown view: Theme → Epic → Story hierarchy (with "No Theme" / "No Epic" sentinel nodes)
  - Backlog view: Sprint → Story flat list (grouped by sprint from sprintSequence)
- switchToBreakdown / switchToBacklog commands registered in extension.ts
- View mode toggle buttons in tree view title bar
- Tree title format: "BACKLOG: Current sprint-N" or "BREAKDOWN: Current sprint-N"

**SprintNode Virtual Tree Nodes**
- New `SprintNode` type (src/types/sprintNode.ts): _kind discriminant, sprintId, label, isBacklog
- BACKLOG_SPRINT_ID sentinel for catch-all node
- isSprintNode() type guard

**BrokenFile Handling**
- New `BrokenFile` type (src/types/brokenFile.ts): broken flag, id, filePath, error, fileType
- Store tracks brokenFiles Map; parse failures stored instead of silently dropped
- Broken files shown under orphan sentinel nodes in tree view

**Drag-and-Drop**
- StoriesDragAndDropController (265 lines):
  - Breakdown: Story → Epic (reassigns epic), Epic → Theme (reassigns theme)
  - Backlog: Story → SprintNode (sprint reassignment + priority 1), Story → Story (insert above)
- BacklogDropHandler (238 lines): priority-based reordering with bump logic
  - Sort guard: drops only valid when sorted by priority ascending
  - updateStorySprintAndPriority(), updateStoryPriorityOnly() in utils
- storiesDragAndDropControllerUtils.ts: frontmatter rewriting utilities

**Sprint Burndown Chart**
- BurndownViewProvider: WebviewView registered below tree in sidebar
  - Renders inline SVG burndown (ideal dashed line vs actual red line)
  - Follows sprint filter; falls back to config.json currentSprint
  - Re-renders on store update, config change, sprint filter change
- burndownUtils.ts (193 lines): pure functions for burndown calculations
  - isBurndownConfigured(), getSprintDateRange(), calculateBurndown()
  - Sprint date ranges derived from firstSprintStartDate + sprintLength
  - Excludes isExcluded stories from ideal/actual lines
- burndownSvgRenderer.ts: renderBurndownHtml(), renderPlaceholderHtml()

**Sort Stories**
- SortService (47 lines): session-only sort state (key + direction), event emitter
- SortKey: 'priority' | 'date' | 'id'; SortDirection: 'asc' | 'desc'
- sortStories command: QuickPick to choose sort order, successive clicks flip direction
- sortStoriesBy(), sortStoriesForTreeView() in storiesProviderUtils.ts
- Tree view refreshes on sort change

**Set Current Sprint**
- setCurrentSprint command: QuickPick from sprintSequence, persists to config.json
- ConfigService.updateCurrentSprint(): reads/writes config.json sprints.current
- Distinct from pickSprint (view-only filter)

**Effort-Based Progress (Story Points)**
- ConfigData: new `storypoints: number[]` (parallel array to sizes, index-aligned)
- getSizePoints(size, sizes, storypoints) utility for point lookups
- StatusBarStats changed from {total, done} to {totalPoints, donePoints}
- Status bar text now shows "pts" suffix
- Default sizes expanded: ['XXS', 'XS', 'S', 'M', 'L', 'XL', 'XXL']
- Default storypoints: [1, 2, 4, 8, 16, 32, 64]

### Config / Schema Changes

- StatusDef: added `isCompletion?: boolean` and `isExcluded?: boolean` flags
- ConfigData: added `sprintLength?: number`, `firstSprintStartDate?: string` (burndown)
- devstories.schema.json: theme prefix, storypoints array, sprint length/start fields, isCompletion/isExcluded on statuses, sizes no longer enum-restricted
